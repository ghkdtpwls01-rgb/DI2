# 빵셀러 데이터 정제·집계 보고서

## 1. 데이터 개요
- 출처: (가상) 빵셀러 베이커리 체인 운영 로그
- 기간: 2025-04-01 ~ 2025-06-30 (3개월)
- 원본 행 수 / 정제 후 행 수: (  2164 )행 → (  1784 )행

## 2. 발견한 품질 문제
- 결측: revenue 2.77%
- 중복: 완전 중복 행 321건
- 이상치: revenue에 -45000
- 표기 혼재: item에 케이크, 케익

## 3. 처리 결정과 근거 (5줄로)
1. 중복 행 (321)건 → 완전 동일 행은 시스템 입력 오류로 간주, 첫 번째만 남기고 제거
2. store_id 공백 → strip()으로 앞뒤 공백 제거해 표기 통일 (같은 매장이 다른 값으로 집계되는 것 방지)
3. item '케익' → '케이크' → replace()로 표기 통일해 같은 상품이 서로 다른 항목으로 집계되는 것 방지
4. revenue 음수 → 별도 보관 (1)건 (환불 등 정상 이벤트일 수 있어 삭제 대신 분리 보관, 매출 분석에서는 제외)
5. revenue 결측 (60)건 → 제거 (이유: 매출 합산이 분석 목적이라 revenue 없는 행은 집계에 포함할 수 없고, 평균으로 채우면 실제로 없었던 매출을 지어내는 왜곡이 발생하므로 행 제거가 적절)

## 4. 주요 KPI 결과 (2줄)
- (월·매장 관점) 가장 매출이 큰 매장과 그 월: S06 매장이 2025-04월에 9,156,000원으로 최고 매출 기록
- (상품 관점) 매출 1위 상품: 쿠키 (총매출 26,690,000원, 평균 단가 6,878원, 365건 판매)

## 5. 한계와 후속 작업
- (예) 환불 데이터의 분기별 발생 원인 분석 필요
- (예) item '케익'/'케이크' 외에 다른 표기 변종이 있는지 추가 점검 필요
- quantity 컬럼의 이상치 비율(14.56%)이 높게 나온 원인 확인 필요 — 실제 이상치인지, IQR 공식이 이 분포에 잘 안 맞는 것인지 추가 검증 필요
- revenue 결측 60건이 특정 매장이나 특정 상품에 몰려 있는지 확인 필요 (무작위 결측이 아니라면 원인 파악 우선)
- 이번 분석은 매장 코드(store_id)만 사용했는데, 실제 매장명·지역 정보가 없어 어느 지역이 매출을 견인하는지는 파악 불가 — 추가 데이터 확보 시 지역별 분석 가능
- store_id별 월별 매출 변동폭이 큰 매장(S05, S06)에 대해 특정 이벤트나 프로모션이 있었는지 등 원인 확인 필요
- 현재는 6~7월 등 일부 데이터만 다루므로, 계절성(여름철 vs 겨울철 판매 패턴 차이) 반영을 위해 1년 치 데이터로 재분석 권장
